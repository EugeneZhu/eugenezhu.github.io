    About

    For this project, I had to design a new feature for an already existing web application. This feature must improve the user experience of the user in some way. I chose to do my project on the instant messaging platform Discord, as I use it often. Below is my design process for this project.

    Brainstorming/Research Phase

    To start off this project, I needed to do some research about Discord itself as well as how people use it. Thus, I conducted interviews during the first week with my fellow friends who also used Discord. Due to the COVID-19 pandemic and lockdown affecting us during this time, I had to conduct interviews virtually, and got feedback through voice call on the Discord app. I asked them about their thoughts on the application as well as what frustrates them most about the platform.

    These interviews were done to help me get a grasp of how interactions between the user and the interface affected their overall experience, and thus I asked the interviewees three specific questions.

    • What is your skill level with the application, or similar sorts of applications?
    • Are there things that frustrate or ease your use of the application?
    • What might you have tried to do that was not clearly possible with the application?

    First Iteration: Feature Creation

    Unfortunately, I did not record the interviews done over voice call. However, I did get plenty of valuable information that I translated into my feature proposals. I found that there were a few similar answers to the questions listed, and thus I created three possible features that I could move on with based on my research. After finalizing the details of each feature, I presented it to my professor as well as my fellow students to get feedback on each feature.

    After presenting it to the class and getting feedback, I ultimately descided to choose to carry on with the third feature, which is a Server Voice Chat Setting.

    Why this feature?

    I made this feature because many of the people I interviewed used the Discord voice chat function constantly, and one major problem they found was that upon joining a voice channel, Discord does not automatically mute their mic. It only mutes the mic if it was already previously muted by the user in their previous voice chat. This is a major problem because it can cause privacy issues for the user, since some people might not be aware their mic is on when talking about something private when they join the voice channel. Thus, the feature I would implement would be a pop-up that shows up everytime a user joins a voice channel. Upon joining the voice channel, Discord will ask the user if they want to join the voice channel with their mic on or off.

    As well, another problem that voice chat users can experience is that they accidentally forget to unmute themselves when talking about something important to the other people in the voice channel, which can affect the experience of the user as well. Thus, I created a voice chat feature that detects speech and sound, and when it does, it give the user a pop-up telling them that Discord has detected speech even though the user is muted. The pop-up would have two options for the user, one of which continues to mute the user's mic, while the other unmutes the mic. Since I know that some users would not like this feature as it might be a privacy issue due to Discord listening to the user's mic, there would also be a setting in the voice settings allowing the user to turn off this feature from being used to give the user peace of mind.

    Second Iteration: User Persona, Final Sketches

    To begin the next phase, I started by creating a user persona for my Discord feature. This user persona helped me determine what kind of user I was designing for and how they would benefit from using my feature. I also kept the user persona as realistic to a real person as possible.

    Once my user persona was created, I presented it to the class and started to create the final mock-up of the feature.

    Third Iteration: Final Mockup

    At this stage of the project, I ran into some problems in creating the mock-ups of the feature. Firstly, since I did not have access to the font Discord uses for it's text, I could not make a completely accurate layout. Thus, I had to use a font that closely resembles the font Discord uses. The second problem I ran into was when I got some people to give my mock-up some feedback. One piece of feedback I got was that having this same pop-up appear every single time a user joins a voice channel would be annoying and would reduce the user experience. Thus, after getting this feedback, I decided to add a "Don't ask again" option that allows the user to remove the pop-up from appearing. The setting would also be available in the Discord voice settings section in case the user wanted to turn it back on.

    Final Thoughts

    Overall, I am satisfied with my feature and final mockup. I think that this would be a valuable feature for Discord to implement as there are times where I also run into the same problems involving voice channels. However, if I had more time to work on this project, I would definitely create a Discord prototype of this feature as well as get some more user feedback and user testing.
